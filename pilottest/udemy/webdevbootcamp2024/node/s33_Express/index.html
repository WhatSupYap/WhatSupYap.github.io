Express로 서버제작하기

우리의 첫 번째 Express 앱

  • Express는 Javascript로 서버 구동하게 해주는 디펜던시다
  • package.json 생성 > 경로이동 > npm init -y (-y 설정값 묻는 절차 스킵)
  • npm i express (구버젼 npm에서 패키지를 package.json에 저잘하려면 --save를 넣어줘야함)
  • 실행후 GET, POST 방식으로 페이지를 호출 해봤다.
  • nodemon으로 서버를 유지해봤다.
  • nodemon = 패키지 설치 명령어
    • npm install -g nodemon
npm install -g nodemon 에서 -g면 글로벌설치 맞죠?
네, 맞습니다. npm install -g nodemon 명령어에서 -g 옵션은 글로벌 설치를 의미합니다. 이 옵션을 사용하면 nodemon이 시스템의 모든 디렉토리에서 사용할 수 있도록 전역적으로 설치됩니다. 따라서 어느 위치에서든지 nodemon을 실행할 수 있게 됩니다. 도움이 필요하시면 언제든지 말씀해 주세요!
그럼 설치파일은 어디에 있고 어떤 방식으로 모든 디렉토리에서 실행 가능한가요?
npm으로 글로벌 설치를 할 때, nodemon과 같은 패키지는 보통 npm의 글로벌 디렉토리에 설치됩니다. 이 디렉토리의 위치는 운영 체제와 npm 설정에 따라 다를 수 있지만, 일반적으로 다음과 같은 경로에 있습니다: Windows: C:\Users\{username}\AppData\Roaming\npm macOS/Linux: /usr/local/lib/node_modules 글로벌 설치된 패키지는 시스템의 PATH 환경 변수에 등록되어 있기 때문에 어느 디렉토리에서든지 실행할 수 있습니다. PATH 환경 변수에는 실행 가능한 파일들의 경로가 포함되어 있어, 쉘(명령 프롬프트나 터미널)이 이 경로들을 참조하여 사용자가 입력한 명령어를 찾아 실행합니다. nodemon을 설치한 후에는 새로운 터미널 창을 열고 nodemon 명령어를 입력하여 어디서든지 실행할 수 있습니다. 만약 PATH에 문제가 있어 실행이 안 된다면, PATH 환경 변수를 확인하고 필요하다면 수정해야 할 수도 있습니다. 도움이 더 필요하시면 알려주세요!
